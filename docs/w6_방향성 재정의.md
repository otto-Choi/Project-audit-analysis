> 예정 기간: 2026년 5월

## 1. 수행 의도

Week 1~5까지의 작업을 통해 다음 구조를 구축하였다.

```
원본 ERP 데이터
  ↓
정제 및 표준화
  ↓
파생변수 생성
  ↓
계정/전표 탐색
  ↓
Excel 출력
```

이번 주 목표는 여기에 **“리스크 탐지 계층(Risk Detection Layer)”** 을 추가하는 것이다.

핵심 방향은 단순 플래그 컬럼 추가가 아니라,
**클라이언트별 리스크 규칙을 config로 관리 가능한 Rule Engine 구조**를 설계하는 데 있다.

이를 통해 프로젝트를:

* 단순 감사 자동화 스크립트가 아니라
* 재사용 가능한 리스크 분석 파이프라인
* ERP 기반 이상거래 탐지 도구

형태로 확장하는 것을 목표로 한다.

---

## 2. 핵심 설계 방향

### 2-1. Rule과 Code의 분리

기존 프로젝트는:

* column_map
* amount_type
* subtotal rule

등을 `config.yaml`로 외부화하였다.

이번 주에는 이상탐지 규칙 또한 config 기반으로 분리한다.

```yaml
anomaly:
  weekend_posting:
    enabled: true

  month_end_posting:
    enabled: true
    last_days: 3

  unusual_round_amount:
    enabled: true
    modulus: 1000000

  large_manual_entry:
    enabled: true
    doc_types: [SA]
    threshold: 100000000
```

목표는:

* 클라이언트마다 다른 리스크 기준 대응
* 노트북 수정 최소화
* Rule 추가 시 코드 수정 범위 축소

이다.

## 4. anomaly_config.yaml 구조 설계

### 4-1. 기본 구조

```yaml
risk_score:
  weights:
    weekend_posting:      1
    month_end_posting:    2
    reversal_entry:       2
    duplicate_journal:    3
    unusual_round_amount: 1
    large_manual_entry:   3

anomaly:

  weekend_posting:
    enabled: true

  month_end_posting:
    enabled: true
    last_days: 3

  reversal_entry:
    enabled: true

  duplicate_journal:
    enabled: true
    key_cols:
      - doc_no
      - line_no

  unusual_round_amount:
    enabled: true
    modulus: 1000000

  large_manual_entry:
    enabled: true
    doc_types: [SA]
    threshold: 100000000
```

## 5. 구현 대상 Rule

### 5-1. weekend_posting

| 항목 | 내용                      |
| -- | ----------------------- |
| 목적 | 비업무일 전기 탐지              |
| 기준 | `_is_weekend == True`   |
| 출력 | `_flag_weekend_posting` |

---

### 5-2. month_end_posting

| 항목     | 내용                        |
| ------ | ------------------------- |
| 목적     | 결산일 집중 전기 탐지              |
| 기준     | 월말 기준 N일 이내               |
| config | `last_days`               |
| 출력     | `_flag_month_end_posting` |

예시:

* 28~31일 전기 집중
* 분기말 집중 전기

---

### 5-3. reversal_entry

| 항목 | 내용                          |
| -- | --------------------------- |
| 목적 | 역분개 전표 식별                   |
| 기준 | `_is_reversal_flag == True` |
| 출력 | `_flag_reversal_entry`      |

---

### 5-4. duplicate_journal

| 항목 | 내용                             |
| -- | ------------------------------ |
| 목적 | 중복 전표 탐지                       |
| 기준 | `(doc_no, line_no)` duplicated |
| 출력 | `_flag_duplicate_journal`      |

---

### 5-5. unusual_round_amount

| 항목     | 내용                            |
| ------ | ----------------------------- |
| 목적     | 비정상 정액 거래 탐지                  |
| 기준     | `abs(_amount) % modulus == 0` |
| config | `modulus`                     |
| 출력     | `_flag_round_amount`          |

예시:

* 10,000,000
* 50,000,000
* 100,000,000

---

### 5-6. large_manual_entry

| 항목     | 내용                         |
| ------ | -------------------------- |
| 목적     | 고액 수동전기 탐지                 |
| 기준     | 특정 전표유형 + threshold 초과     |
| config | `doc_types`, `threshold`   |
| 출력     | `_flag_large_manual_entry` |

---

## 6. Risk Score 설계

### 6-1. 목적

단순 flag 집합이 아니라,
감사인이 우선 검토해야 할 거래를 정렬하기 위한 위험도 점수를 생성한다.

---

### 6-2. 계산 방식

```python
risk_score =
    weekend_posting      * 1 +
    month_end_posting    * 2 +
    reversal_entry       * 2 +
    duplicate_journal    * 3 +
    unusual_round_amount * 1 +
    large_manual_entry   * 3
```

---

### 6-3. 출력 컬럼

| 컬럼            | 설명                  |
| ------------- | ------------------- |
| `_risk_score` | 위험 점수               |
| `_risk_level` | LOW / MEDIUM / HIGH |
| `_risk_flags` | 탐지된 Rule 문자열        |

예시:

| doc_no     | risk_score | risk_level | risk_flags                |
| ---------- | ---------- | ---------- | ------------------------- |
| 5100015498 | 6          | HIGH       | reversal;month_end;manual |
