## 2. SAP 데이터셋 구조 파악

### 데이터셋 내 전체 테이블 목록 검토
데이터셋에는 약 70여 개의 SAP 테이블이 포함되어 있다. 감사 분석 목적에 맞게 필요한 테이블만 선별하는 과정을 거쳤다.

**제외한 영역**
- 주소 관련 (adr6, adrc 등) → 감사 분석 불필요
- 구매 관련 (ekko, ekpo, ekbe 등) → MM 모듈, 분석 범위 밖
- 판매 관련 (vbak, vbap, vbrk 등) → SD 모듈, 분석 범위 밖
- 환율 관련 (tcurr 등) → bseg에 이미 기능통화 환산금액(dmbtr) 존재
- 세금 코드, 측정단위 등 코드성 테이블

### 환율 처리 결정
bseg 테이블에 `dmbtr`(현지통화 환산금액)과 `wrbtr`(거래통화 원래금액)이 모두 존재한다. 즉 외화 거래도 이미 기능통화로 환산된 값이 있으므로 별도 환율 테이블(tcurr) 없이 `dmbtr`만 사용하면 된다. `waers`(통화코드)는 외화 거래 식별 목적으로 함께 보존한다.

## 3. 사용 테이블 확정 (7개)

| 테이블 | SAP 원래 명칭 | 용도 | 비고 |
|--------|--------------|------|------|
| **bkpf** | Accounting Document Header | 전표 헤더 | 날짜, 문서유형, 역분개 여부 |
| **bseg** | Accounting Document Segment | 전표 라인 | 금액, 계정, 차대변, 거래처코드 |
| **ska1** | G/L Account Master | 계정 마스터 | BS/PL 구분(xbilk) |
| **skat** | G/L Account Master Text | 계정명 텍스트 | txt50 사용, spras='E' 필터 필수 |
| **kna1** | Customer Master | 고객(매출처) 마스터 | kunnr → 고객명 |
| **lfa1** | Vendor Master | 벤더(매입처) 마스터 | lifnr → 벤더명 |
| **cskt** | Cost Center Text | 원가센터명 | kostl → 원가센터명 |

### 조인 구조
```
bkpf ─┐
      ├─ belnr + gjahr + bukrs 기준 조인 → 원장 기본 뼈대
bseg ─┘
      ├─ saknr → ska1 + skat(spras='E')  계정코드 → 계정명, BS/PL 구분
      ├─ kunnr → kna1                    고객코드 → 고객명
      ├─ lifnr → lfa1                    벤더코드 → 벤더명
      └─ kostl → cskt                    원가센터코드 → 원가센터명
```

## 4. 테이블별 사용 컬럼 확정

### bkpf (전표 헤더)
```
bukrs       회사코드
belnr       전표번호 (문자열 처리 필수 — 엑셀 지수표기 문제)
gjahr       회계연도
blart       문서유형 (SA=수동전기, RE=매입전표 등)
bldat       문서일
budat       전기일
monat       회계기간(월)
waers       통화코드
stblg       역분개 참조전표번호
xreversal   역분개 여부 플래그
bktxt       전표 헤더 텍스트
```

**제외 컬럼**: cpudt/cputm(입력일시), usnam(전기자 ID), tcode(T-code), 환율 관련, 결재 관련, 카드번호 등  
→ 계정담당자 수준에서 받을 수 없는 컬럼들

### bseg (전표 라인)
```
buzei       라인번호
shkzg       차대변 구분 (S=차변, H=대변) ← C형 플래그로 활용
saknr       계정코드
dmbtr       현지통화금액 ← 분석에 사용
wrbtr       거래통화금액 ← 외화 거래 식별용으로 보존
sgtxt       적요
kunnr       고객코드
lifnr       벤더코드
kostl       원가센터
gsber       사업영역
prctr       손익센터
```

### ska1 (계정 마스터)
```
saknr       계정코드
xbilk       BS/PL 구분 (X=BS, 공백=PL)
```
→ 계정명은 없음, skat에서 별도 조인 필요

### skat (계정명 텍스트)
```
spras       언어코드 ← 'E'(영어)만 필터링
saknr       계정코드
txt50       계정명 전체 ← 사용
```
→ 다국어 데이터이므로 spras='E' 필터링 필수

### kna1 / lfa1 (거래처 마스터)
→ 컬럼 확인 후 고객명/벤더명 컬럼 특정 예정

### cskt (원가센터 텍스트)
→ 컬럼 확인 후 원가센터명 컬럼 특정 예정

## 5. 주요 기술적 발견사항

**belnr 지수표기 문제**
SAP 전표번호(belnr)는 10자리 숫자다. 엑셀이 자동으로 `5.11E+09`와 같은 지수 표기법으로 변환한다. CSV 저장 및 pandas 읽기 시 반드시 문자열(str)로 처리해야 한다.

```python
df = pd.read_csv('bkpf.csv', dtype={'belnr': str})
```

**계정코드 앞자리 0 보존**
ska1의 saknr 컬럼에 `0000500020` 형태로 앞자리 0이 포함되어 있다. 숫자로 읽으면 앞자리 0이 소실되므로 문자열로 처리해야 한다.

**skat 다국어 중복**
같은 계정코드에 언어별로 여러 행이 존재한다. `spras='E'` 필터링 없이 조인하면 계정명이 중복 생성된다.

**통화 처리**
- 기능통화 환산금액(dmbtr)이 이미 존재하므로 환율 테이블 불필요
- `waers ≠ 기능통화`인 행을 외화 거래로 식별 가능
