## 2. Rule Engine 구조

### 2-1. 처리 흐름

```
master_gl_clean.csv
  ↓
anomaly_config.yaml 로드
  ↓
Rule loop 실행 → _flag_* 컬럼 생성
  ↓
_risk_score 계산 (flag × 가중치 합산)
  ↓
_risk_level 분류 (LOW / MEDIUM / HIGH / NONE)
  ↓
_risk_flags 생성 (탐지된 Rule 한국어 문자열)
  ↓
master_gl_anomaly.csv / anomaly_summary.csv 저장
```

### 2-2. anomaly.py 함수 구성

| 함수 | 역할 |
|---|---|
| `load_anomaly_config()` | yaml 로드 |
| `apply_rules()` | Rule loop 실행, `_flag_*` 생성 |
| `calculate_risk_score()` | 가중치 합산, level·flags 생성 |
| `summarize_anomalies()` | Rule별 건수·비율·평균 금액 집계 |
| `run_anomaly_pipeline()` | 전체 파이프라인 단일 호출 진입점 |

### Rule별 탐지 방식

| Rule | 탐지 기준 | config 파라미터 |
|---|---|---|
| 비업무일 전기 | `post_date` 요일이 토·일 | — |
| 월말 집중 전기 | 월말 기준 마지막 N일 이내 전기 | `last_days: 3` |
| 역분개 전표 | `_is_reversal_flag == True` | — |
| 중복 전표 | `(doc_no, line_no)` 중복 | `key_cols` |
| 비정상 정액 거래 | `abs(_amount) % modulus == 0` | `modulus: 1,000,000` |
| 고액 수동 전기 | 특정 전표유형 + threshold 초과 | `doc_types`, `threshold` |
| 차대불균형 전표 | 전표 내 차변합계 ≠ 대변합계 | `tolerance: 0.01` |
| 계정별 이상금액 | 계정별 Z-score \|z\| > threshold | `z_threshold: 3.0`, `min_count: 10` |

## 5. Streamlit 변경사항

### 탭 구조 도입

```
기존: 단일 화면 (계정 탐색)
변경: [계정 탐색] [Risk View] 2탭 구조
```

### [계정 탐색] 탭 — 전표 조회 개선

- **통합 분개 테이블 추가**: 차변·대변 라인을 한 테이블에 통합, 차변금액·대변금액 컬럼 나란히 표시
- **차대 차이 지표 추가**: 차변 합계 / 대변 합계 / 차대 차이 metric 카드
- 계정 드릴다운 버튼은 테이블 아래에 유지

### [Risk View] 탭 — 신규

**Sidebar 필터**

| 항목 | 방식 |
|---|---|
| 최소 Risk Score | slider (0 ~ max) |
| Risk Level | selectbox (ALL / HIGH / MEDIUM / LOW) |
| Flag 종류 | multiselect (한국어, AND 조건) |

**메인 화면**

- 전체 건수 / 필터 결과 / HIGH 건수 / MEDIUM 건수 요약 지표
- Rule별 탐지 현황 expander (건수·비율·평균 금액)
- High Risk Transactions 테이블 (Score 내림차순)
- 행 선택 시 전표 상세 인라인 표시

## 6. 설계 결정 사항

| 결정 | 이유 |
|---|---|
| config 범위를 파라미터화로 제한 | dynamic expression·DSL은 현 단계 과설계 |
| Rule 단위: line, Score 단위: line (doc 집계 없음) | Streamlit 필터·drill-down 연계 단순화 |
| 구버전 `_flag_*` 컬럼을 UI에서 제외 | `_flag_weekend` 등 w3~w5 산출물과 중복 탐지 방지 |
| Z-score 계정별 적용 + min_count 조건 | 표본 부족 계정의 오탐 방지 |
| 전표 조회에 통합 테이블 + 드릴다운 버튼 병존 | 가독성(테이블)과 탐색성(버튼) 동시 확보 |
