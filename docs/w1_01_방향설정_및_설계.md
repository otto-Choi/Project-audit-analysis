# Project-audit-analysis

> **이 프로젝트는 외부감사 실무에서 감사인 관점의 데이터 분석 역량을 기르기 위해 개인적으로 진행하는 프로젝트입니다.**  
> 회사가 제시하는 자료(원장, BS, PL)를 Python으로 가공·분석하는 파이프라인을 구축하고, 분석적 절차·이상치 탐지·드릴다운 탐색 기능을 개발하는 것을 목표로 합니다.  
> 수행 기간: 2026년 ~ (진행 중)  
> 파이썬 학습을 겸하여 실무에 즉시 투입 가능한 로컬 분석 도구를 구축하는 개인 프로젝트입니다.

---

## 개요

외부감사에서 감사인은 감사대상기업으로부터 다양한 형태의 자료를 수령하고 이를 분석하여 감사 절차를 수행한다. 그러나 기업마다 데이터 형태가 완전히 상이하고, 회계법인의 보안 규범상 외부 서비스를 활용하기 어렵다는 제약이 존재한다.

이 프로젝트는 1) **다양한 형태의 회사 제시 자료를 일관된 구조로 정제하는 파이프라인**과 2) **분석적 절차, 이상치 탐지, 계정 간 드릴다운 탐색 기능**을 로컬 환경에서 구축하는 것을 목표로 한다.

---

## 프로젝트 목적

- 감사대상기업별로 상이한 원장·BS·PL 데이터를 **일관된 표준 스키마로 정제**하는 파이프라인 구축
- 정제된 데이터를 바탕으로 **분석적 절차(증감분석, 추세분석, 합리성 검토)** 를 자동화
- 비업무시간 전기, 역분개, 특수관계자 거래 등 **이상치를 자동으로 탐지하고 플래그**하는 기능 개발
- 매출 원장에서 분개를 확인하고, 분개에서 관련 계정 원장으로 이동하는 **계정 간 드릴다운 탐색** 기능 구현
- 조서 작성 시 XLOOKUP/SUMIFS로 바로 참조할 수 있는 **표준화된 참조 데이터 엑셀 자동 생성**
- 회계법인 보안 환경(인터넷 차단, 외부 전송 불가)에서 **완전 로컬로 실행 가능한 도구** 구축

## 프로젝트 진행 방식

- **분석 환경**
  - Python 기반 Jupyter Notebook에서 개발·학습 진행
  - Streamlit으로 로컬 탐색 UI 구현 (드릴다운 기능)
  - 핵심 라이브러리: pandas, openpyxl, matplotlib, streamlit

- **데이터**
  - 연습용 샘플: Kaggle 공개 데이터셋 (irfansharif/generalledger 등)
  - 실제 적용 대상: 감사 업무에서 수령하는 회사 제시 자료 (원장, BS, PL)
  - 실제 클라이언트 데이터는 보안상 레포지토리에 포함하지 않음

- **개발 구조**
  - 클라이언트별 설정은 config 파일로 관리, 분석 로직은 재사용 가능하도록 분리
  - 변환(xlsx→csv) / 정제(파생변수 생성) / 분석의 3단계 파이프라인
  - 원본 데이터는 절대 수정하지 않는 원칙 준수

- **설계 결정 과정**
  - 코드 작성 전 충분한 설계 과정을 거쳐 구조를 확정
  - 각 설계 결정의 이유를 문서로 기록 → [DESIGN.md](./DESIGN.md) 참조
  - 주차별 진행 기록은 [docs/](./docs/) 폴더에 정리

## 주요 기능 및 구현 내용

### 1. 변환 모듈 (xlsx → csv)

- 회사가 제출한 엑셀 파일을 분석 가능한 CSV로 변환
- 클라이언트별 1회 설정 후 재사용 (수정본·다른 계정 보조부 수령 시 동일 설정 적용)
- 병합 셀 ffill 처리, 숨긴 행/열 제거, 헤더 위치 자동 조정, 계정코드 앞자리 0 보존 등 처리

### 2. 정제 모듈 (파생변수 생성)

- 다양한 형태의 날짜(구분자형, 8자리 숫자, 엑셀 시리얼, 한글형)를 표준 datetime으로 통일
- 다양한 금액 형태(차대변 분리, 부호형, 플래그형, 잔액 누계형)를 차변/대변으로 표준화
- 소계행 자동 탐지 및 분리 (키워드, 계정코드 자릿수, 필수 컬럼 공백 기준)
- 계정 마스터·거래처 마스터와 조인하여 파생변수 자동 생성
  - 날짜 분해(연·월·일·회계기간·주말여부), 금액 표준화, 계정 계층구조, 비용 구분, 특수관계자 여부, 이상치 플래그 등

### 3. 참조 데이터 엑셀 자동 생성

- 조서에서 XLOOKUP/SUMIFS로 바로 참조할 수 있는 표준화된 엑셀 파일 자동 생성
- 시트 구성: BS 원본 / PL 원본 / GL 원본 / GL 가공 / TB(시산표) / 수식모음
- GL 가공 시트의 파생변수는 A열부터 고정 배치 → 수식이 클라이언트 불문 항상 동일
- 수식모음 시트에 검증 수식(원본 vs 가공 일치 여부) 및 분석 수식 자동 포함

### 4. 이상치 탐지 모듈

- 비업무시간(주말·새벽) 전기 탐지
- 역분개·취소 전표 식별
- 특수관계자 거래 플래그
- 수동 전기 탐지
- 기간 외 전기 탐지
- 중복 전표 탐지

### 5. Streamlit 드릴다운 탐색 UI

- 계정 선택 → 원장 조회 → 전표 클릭 → 분개 확인 → 관련 계정 이동의 드릴다운 탐색
- 수익 인식 적절성 확인 등 회계처리 이해 목적의 계정 간 추적
- 완전 로컬 실행, 인터넷 불필요

## 설계 원칙

- **원본 불변**: 회사가 제출한 원본 파일은 절대 수정하지 않음 (`_idx` 컬럼 추가만 예외)
- **config 기반 유연성**: 클라이언트별 설정은 config 파일로 관리, 분석 로직은 재사용
- **완전 로컬**: 모든 처리는 로컬에서 완결, 외부 전송 없음
- **역할 분리**: 조서 작성은 감사인 영역, Python은 분석·탐색까지만
- **검증 가능성**: 모든 가공 과정은 로그와 검증 수식으로 추적 가능
- **버전 관리**: 수령일 기준 폴더 구조로 수정본 수령 이력 관리
