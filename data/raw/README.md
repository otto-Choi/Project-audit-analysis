# 📂 Raw Data Sample & Dataset Overview

본 폴더는 감사 데이터 분석 파이프라인의 테스트를 위한 **샘플 데이터셋**을 포함하고 있습니다. 용량 제한 및 저작권 준수를 위해 전체 데이터 중 일부(Sample)만을 추출하여 업로드하였습니다.

## 1. 데이터셋 선정 배경 및 목적
실제 감사 현장에서 수령하는 데이터는 기업의 ERP(SAP 등)에 따라 그 형태가 매우 다양합니다. 본 프로젝트는 다음과 같은 이유로 **Kaggle의 SAP FI Dataset**을 기반으로 샘플을 재구성하였습니다.

* **현실성**: 실제 SAP FI 모듈의 핵심 테이블(`BKPF`, `BSEG`, `SKA1`, `SKAT` 등) 구조를 그대로 유지하여 실무와 유사한 분석 환경을 제공합니다.
* **범용성**: 특정 ML 모델을 위해 가공된 데이터가 아닌, 원천 테이블 형태를 사용하여 전처리(Cleaning)와 병합(Join) 과정을 직접 구현할 수 있습니다.
* **훈련 목적**: 전표 수준의 상세 데이터와 다국어/다통화 환경을 포함하고 있어, 감사인의 관점에서 이상치를 탐색하는 훈련에 최적화되어 있습니다.

## 2. 샘플 데이터 파일 설명
업로드된 샘플은 `01_mearge.ipynb` 단계를 거쳐 생성된 중간 산출물(Master GL)의 일부입니다.

| 파일명 | 데이터 형태 (Type) | 주요 특징 |
| :--- | :--- | :--- |
| **`master_gl_절대금액 표시.csv`** | **C형 (Absolute)** | 현지통화금액이 절대값으로 표시되며, `차대구분` 컬럼으로 성격(S/H)을 분류하는 형태입니다. |
| **`master_gl_순금액 표시.csv`** | **B형 (Net)** | 별도의 구분자 없이 차변(+), 대변(-) 부호로 금액의 성격을 표시하는 형태입니다. |

## 3. 주요 스키마 (Standard Schema)
정제 모듈을 통해 표준화되는 주요 필드는 다음과 같습니다.
* `doc_no` (전표번호): 전표 식별을 위한 키 값 (String 유지)
* `acc_code` (계정코드): 계정 과목 코드
* `post_date` (전기일): 회계상 거래가 기록된 날짜
* `amount` (금액): 분석의 대상이 되는 핵심 수치
* `dc_indicator` (차대구분): 전표의 차변/대변 성격

## 4. 데이터 제한 및 저작권 안내
* **샘플링**: 본 저장소에 업로드된 파일은 전체 로우 데이터 중 로직 검증에 필요한 최소한의 행(Row)만을 포함하고 있습니다.
* **저작권**: 원본 데이터는 Kaggle의 `mustafakeser4/sap-dataset-bigquery-dataset`를 기반으로 하며, 분석 연습용으로만 사용됩니다.
* **개인정보**: 본 데이터는 합성 데이터(Synthetic Data)로, 실제 특정 기업의 기밀이나 개인정보를 포함하고 있지 않습니다.
