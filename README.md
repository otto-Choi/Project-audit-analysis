# Project-audit-analysis

# SAP General Ledger 기반 감사 분석 도구

## 프로젝트 개요

본 프로젝트는 SAP에서 추출한 전표(General Ledger) 데이터를 정제·분석하여, 감사인이 코드 없이도 계정별 드릴다운 탐색과 이상거래 식별을 수행할 수 있는 도구를 구축하는 것을 목표로 합니다. 2026년 4월부터 5월까지 주차별 스터디 프로젝트로 진행되었으며, 데이터 병합부터 인터랙티브 탐색 UI까지 단계적으로 구현했습니다.

## 프로젝트 목적 및 의의

감사 실무에서 SAP GL 데이터는 대개 7~10개의 분리된 테이블로 제공되며, 이를 전표 단위로 재구성하는 것만으로도 상당한 수작업이 요구됩니다. 본 프로젝트는 이 병합-정제 과정을 파이프라인으로 자동화하고, 결과를 드릴다운 방식으로 탐색할 수 있는 UI를 제공함으로써 다음 질문에 답하고자 했습니다.

> "Python을 모르는 감사인도 대용량 전표 데이터를 구조적으로 탐색할 수 있는가?"

특히 계정 계층을 SAP 계정코드의 Prefix 구조에서 자동 생성하는 방식은, 실제 감사 현장에서 완전한 계정과목표가 제공되지 않는 경우에 대응하기 위한 것입니다. 또한 Streamlit UI는 감사인이 Excel에서 수행하는 계정 간 교차 참조 흐름을 그대로 재현하도록 설계되었습니다.

## 분석 방법

**데이터 파이프라인:** 병합(01) → 정제(02) → 탐색(03)의 3단계로 구성됩니다. 정제 단계는 YAML 설정 파일로 컬럼 매핑을 관리하며, SAP 수출 포맷의 세 가지 유형(절대금액형·순금액형·차대분리형)을 모두 지원합니다. 정제 결과물에는 원본 18개 컬럼 외에 회계연월, 요일, 거래 부호 등 파생변수 16개가 추가됩니다.

**이상거래 플래그:** 정제 단계에서 6개의 플래그 변수를 부착합니다. 주말·휴일 전기, 단기 역분개, 회계기간 외 전기 등을 탐지하며, 최종 판단이 아닌 심층 검토 후보를 선별하는 데 목적이 있습니다.

**계정 계층 생성:** 계정코드 Prefix를 기준으로 대분류·중분류를 자동 생성한 뒤, 한글 계정명을 수동으로 보완하는 방식을 택했습니다. 완전한 계정과목표에 의존하지 않으므로, 계정 체계가 다른 클라이언트에도 일정 수준 적용 가능합니다.

**Streamlit UI:** 사이드바에서 대/중/세분류 순으로 계정을 선택하고, 원장 테이블의 행을 클릭하면 해당 전표의 분개가 펼쳐지며, 분개 내 상대 계정을 클릭하면 그 계정의 원장으로 이동합니다. 감사인이 Excel에서 수행하는 교차 추적 흐름을 UI 내에서 재현한 것입니다.

## 주요 결과

Week 4 기준, 파이프라인 전 단계가 샘플 데이터에서 정상 작동합니다. 정제 모듈은 세 가지 입력 포맷을 정확히 처리하고, Streamlit UI는 대분류 선택부터 전표 분개 조회까지의 전체 드릴다운 경로를 지원합니다. Prefix 기반 계층 자동 생성은 예상보다 안정적으로 작동하여, 수동 수정이 필요한 계정이 소수에 그쳤습니다.

## 한계

가장 큰 한계는 단일 샘플 데이터셋으로만 검증되었다는 점입니다. 정제 설정과 컬럼 매핑은 해당 샘플 구조에 맞춰져 있으며, 다른 클라이언트의 SAP 수출 파일에 적용 시 재설정이 얼마나 필요한지는 확인되지 않았습니다. 순금액형(B형) 입력 경로는 설계는 완료되었으나 실제 데이터로 검증하지 못했습니다.

이상거래 플래그는 단순 규칙 기반으로 설계되어, 주말 전기나 정액 거래가 정상인 업종에서는 위양성이 높을 수 있습니다. 보다 통계적 근거를 가진 Benford's Law 분석은 계획 단계에 있으며 아직 구현되지 않았습니다.

또한 현재 도구는 탐색 단계에서 멈춥니다. 감사 조서, 발견 사항 요약, 검토 이력 등 감사 산출물을 생성하는 기능이 없어, 실무 활용 범위가 사전 분석 단계로 제한됩니다.

## 향후 과제

우선순위 순으로, Benford's Law 분석 모듈 구현, 감사 조서용 Excel 자동 생성(XLOOKUP/SUMIFS 수식 포함), B형 입력 실데이터 검증 순으로 진행할 예정입니다. 장기적으로는 구조가 다른 제2의 데이터셋에 파이프라인 전체를 적용해보는 것이 도구의 범용성을 검증하는 가장 의미 있는 테스트가 될 것입니다.
